Configuración Avanzada

Una vez que domines lo básico, querrás personalizar Docusaurus para que se adapte perfectamente a tus necesidades.

Personalización de la Barra Lateral (sidebars.ts)​

El archivo sidebars.ts controla la navegación izquierda de tu documentación.

Sidebar Automático​

La configuración por defecto suele generar la barra lateral automáticamente basándose en la estructura de carpetas:

const sidebars: SidebarsConfig = {
  tutorialSidebar: [{type: 'autogenerated', dirName: '.'}],
};

Sidebar Manual​

Puedes definir explícitamente el orden y qué mostrar:

const sidebars: SidebarsConfig = {
  tutorialSidebar: [
    'intro', // ID del documento
    {
      type: 'category',
      label: 'Guías de Inicio',
      items: ['tutorial-basics/create-a-document'],
    },
    {
      type: 'link',
      label: 'Docusaurus Web',
      href: 'https://docusaurus.io',
    },
  ],
};

Plugins​

Docusaurus es altamente modular. Puedes añadir funcionalidades extra mediante plugins.

Instalar un Plugin​

Por ejemplo, para añadir búsqueda local (si no usas Algolia):

  1. Instala el plugin:

    bun add @easyops-cn/docusaurus-search-local

  2. Agrégalo a docusaurus.config.ts:

    const config: Config = {
      // ...
      plugins: [
        [
          require.resolve("@easyops-cn/docusaurus-search-local"),
          {
            hashed: true,
          },
        ],
      ],
      // ...
    };

Swizzling (Personalización Profunda)​

"Swizzling" es el término de Docusaurus para permitirte copiar un componente interno del tema a tu carpeta src/theme para modificarlo.

Precaución

El swizzling puede romper la compatibilidad con futuras actualizaciones de Docusaurus si la estructura interna cambia. Úsalo solo si es necesario.

Para ver qué componentes puedes personalizar:

bun run swizzle --list

Para extraer un componente (por ejemplo, el Footer):

bun run swizzle @docusaurus/theme-classic Footer -- --eject

Esto copiará el código fuente del Footer a src/theme/Footer. Ahora puedes editarlo como cualquier componente de React.

i18n (Internacionalización)​

Docusaurus tiene soporte integrado para múltiples idiomas.

  1. Configura los idiomas en docusaurus.config.ts:

    i18n: {
      defaultLocale: 'es',
      locales: ['es', 'en'],
    },

  2. Ejecuta los comandos para copiar tus archivos de documentación a las carpetas de traducción:

    bun run write-translations
    mkdir -p i18n/en/docusaurus-plugin-content-docs/current
    cp -r docs/** i18n/en/docusaurus-plugin-content-docs/current/
